# 1. Покупаем VPS

Заказываем 2 виртуальных сервера:
1. В России
2. За границей - Голландия, Финляндия или в другой стране, с которой у нас будет минимальный пинг

В качестве операционной системы лучше всего выбрать Debian или Ubuntu.

# 2. Первоначальная настройка серверов

Подключаемся к серверу по SSH (порт 22) и через терминал производим настройку каждого сервера.

```
apt update
```

```
sudo apt upgrade
```

> [!info] 
>  Если команда **sudo** не найдена, то сначала выполняем следующие команды:
>  ```
>  su -
>  apt install sudo
>  ```

Выбираем пункт "install the package maintainer's version" `Enter`
![[Configuring openssh-server.png]]

```sh title:"Открываем файл sshd_config:"
sudo nano /etc/ssh/sshd_config
```

> [!info] 
>  Если редактор **nano** не установлен, то устанавливаем следующей командой:
>  ```
>  sudo apt install nano
>  ```

Находим в тексте строчку \#Port 22. Стираем символ решётки и меняем порт на любой другой, но не больше, чем **65535**. 

Сохраняем изменения: `Ctrl + O`, `Enter`
Выходим из редактора: `Ctrl + X`

```sh title:"Перезапускаем службу SSH:"
sudo systemctl restart sshd
sudo systemctl restart ssh (для Debian/Ubuntu)
```

```sh title:"Перезапускаем сервер:"
reboot
```

Подключаемся к серверу по SSH, указав в настройках подключения новый порт.

# 4. Настраиваем доступ к панели по SSH туннелю для уменьшения рисков её блокировки третьими лицами (для обоих серверов)

Если есть доступ к панели через браузер, то заходим в панель. В настройках в поле "IP-адрес для управления панелью" вводим IP: `127.0.0.1`

![[IP управления панелью.png]]

Сохраняем изменения и перезапускаем панель. В результате этих изменений панель перестанет быть доступна из интернета.

> [!info] 
> Если доступа к панели изначально не было, то IP-адрес для управления панелью можно изменить через терминал по SSH, используя следующие команды:
> 
> Переходим в корневой каталог:
> ```
> cd /
> ```
> Задаём прослушивание только на 127.0.0.1:
> ```
> /usr/local/x-ui/x-ui setting -listenIP 127.0.0.1
> ``` 
> 
> Перезапускаем службу 3x-ui:
> ```
> systemctl restart x-ui
> ```
> 
> Проверяем настройку сокетов и смотрим что бы встал сокет к процессу 3x-ui из локальной сети. Должно получиться так -> 127.0.0.1:*Номер порта нашей панели*
> ```
> ss -ntpl
> ```

> [!hint] 
> Для управления панелью из терминала нужно ввести команду и выбрать нужный пункт:
> ```
> x-ui
> ``` 

Теперь панель закрыта для доступа “в интернет”, а доступна только в локальной сети.

# 5. Поиск безопасного сайта-донора (SNI) с помощью утилиты Reality-TLS-Scanner  (для обоих серверов)

Для маскировки VPN-соединения под трафик обычного сайта нам необходимо найти надежный сайт в подсети своего хостера, чтобы улучшить маскировку и скорость работы. Для этого воспользуемся утилитой [Reality-TLS-Scanner](https://github.com/XTLS/RealiTLScanner).

1. Используя IP русского сервера мы ищем надежный работающий сайт в зонах RU или SU (RU предпочтительнее).
2. Используя IP зарубежного сервера мы ищем надежный незаблокированный РКН сайт в зонах COM, ORG и т.д.

Создаем папку Reality-TLS-Scanner и копируем в нее файл RealiTLScanner-windows-64.exe.

Далее нам нужно запустить консоль в папке с программой. Для этого заходим в проводнике в папку с утилитой и прописываем `cmd` прямо вместо пути в верхней части проводника и нажимаем `[Enter]`.

![[Проводник.png]]

Если мы используем Total Commander, то `cmd` нужно прописать в командной строке внизу.

![[Total Commander.png]]

В консоли вводим следующую команду:
`RealiTLScanner-windows-64 -addr 00.00.00.00 -out file.txt`, где вместо нулей нужно прописать IP нашего сервера.

Например: `RealiTLScanner-windows-64 -addr 84.143.201.156 -out file.txt`

```sh
RealiTLScanner-windows-64 -addr 84.143.201.156 -out file.txt
```

![[Консоль.png]]

После этого Reali-TLS-canner начнет бесконечно сканировать всю подсеть нашего хостинга и складывать результаты в файл `file.txt`.

Не закрывая консоль, периодически проверяем `file.txt` и ищем подходящий сайт.  Предложенные сайты открываем в браузере и оцениваем его с точки зрения надежности. Лучше всего подойдут сайты учреждений, магазинов, компаний, банков. Сайт уже должен иметь историю существования и не быть однодневкой. Сайты различных казино, лотерей и другой около законной или спорной тематики нам не подойдут, так как в любой момент могут быть заблокированы или закрыты. Также сайт должен иметь заглавную страницу, не переадресовывающую на какой-нибудь другой домен. В идеале, чтобы переадресации с главной страницы вообще не было.

> [!note] 
> Например сканер нашел в нашей подсети **sberbank.ru**. Казалось бы идеальный вариант, но если перейти по этой ссылке, то нас переадресует на страницу sberbank.ru/ru/person. Лучше избегать таких сайтов и поискать другой.

> [!attention] 
> Сайт должен поддерживать подключения по TLS v.1.3 и HTTP/2. В файле результатов в строке найденного сайта после IP должно быть указано `TLS 1.3,h2`
> 
> ![[TLS.png]]

В случае с нашим зарубежным сервером, хорошим вариантом из полученного списка можно выделить *github.com* или *yahoo.com*.

![[TLS2.png]]

Поле { Протокол } -> выбираем **vless**.
Поле { Порт } -> обязательно пишем **==8443==**.

Пропускаем

## 6.3 Вкладка "Поток"

Если мы реализовали пункт [[#4. Настраиваем доступ к панели по SSH туннелю для уменьшения рисков её блокировки третьими лицами]], то для того чтобы в будущем создавались рабочие конфигурации c IP сервера, а не с localhost нужно настроить External Proxy. Выбираем **Reality**.

Поле { uTLS } -> выбираем под какой браузер будем маскироваться, например **firefox**.
Поле { Цель } -> пишем сайт, который выбрали в пункте [[#5. Поиск безопасного сайта-донора (SNI) с помощью утилиты Reality-TLS-Scanner]]. Например: **www\.dzen.ru:433**
Поле { SNI } -> пишем сайт с www и без. Например: **www\.dzen.ru, dzen.ru

Чуть ниже нажимаем кнопку \[ Получить новый сертификат ].

![[Безопасность.png]]

Нажимаем "Включить".

Проверяем, чтобы были включены пункты ✓︎ HTTP, ✓︎ TLS, ✓︎ QUIC, ✓︎ FAKEDNS.
Пункты { Только метаданные } и { Только маршрутизация должны } быть **ВЫКЛЮЧЕНЫ**.

Далее внизу нажимаем кнопку \[ Создать ].

![[Сниффинг.png]]

Поле { Привязанные входящие } -> выбираем наше  соединение, которое мы создали ранее.

После выбора соединения вверху появится поле { Flow }.
В нем выбираем пункт "**xtls-rprx-vision**".

Нажимаем кнопку \[ Сохранить ].

> [!info]

> Сохраняем изменения, перезапускаем панель и обновляем страницу

## 6.7 Проверяем работу VPN

Качаем клиент подключения к VPN для нашей системы. Список клиентов для разных клиентов можно найти странице [Приложения для VPN](https://forms.gle/aNhc1SgG8H6BjzT58). Рекомендую приложения Happ или V2RayTUN.

Нажимаем на странице панели пункт меню \[ Клиенты ] и напротив нужного нам клиента нажимаем значок QR-кода. Появится QR-код. Нажимаем на него левой кнопкой мыши. Ссылка конфигурации скопирована в буфер обмена. Вставляем в приложении VPN эту ссылку и запускаем VPN соединение.

Заходим на сайт [2ip.ru](https://2ip.ru) и смотрим наш IP, если он совпадает с IP нашего сервера, то VPN работает.
